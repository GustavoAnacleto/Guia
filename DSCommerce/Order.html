Order class

Criamos a class Order e iniciamos o relacionamento entre (Order & User) esse relacionamento é um para muitos sendo um usuário pode ter vários pedidos.

  • @ManyToOne: indica que o relacionamento é para um
  • @JoinColumn(name = "client_id"): cria uma coluna na tabela tb-order que servira de chave estrangeira para relacionar com User.
No Spring Boot, ao implementar uma entidade em seu projeto, se ela tem um relacionamento para um, geralmente você já implementa o relacionamento. Isso significa que, ao definir suas entidades no Spring Boot, você pode utilizar anotações como @OneToOne, @ManyToOne, ou @JoinColumn para representar os relacionamentos entre as entidades. Isso permite que o Spring Boot faça a configuração do mapeamento entre as tabelas do banco de dados automaticamente, facilitando o desenvolvimento e reduzindo a necessidade de escrever consultas SQL manualmente.

Na imagem acima, é mostrada a classe Order e, ao final da imagem, temos:
@ManyToOne: que significa muitos para um.
@JoinColumn: que significa juntar coluna.
Com isso, no banco de dados do projeto é criada uma coluna chamada client_id, que faz referência ao atributo client da classe User.
Isso quer dizer que, quando um cliente cria um pedido, o sistema automaticamente vincula um número chamado id daquele pedido com aquele cliente. Esse número é único em todo o sistema e não se repete para outro pedido.
